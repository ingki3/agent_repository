# Agent Client — PRD

## 1. Vision & Problem
### 1.1 Vision
Agent Client는 1인 운영자가 모바일에서 Simple Claw, Open Claw 등 다중 에이전트와 Telegram 호환 UX로 대화하고 작업을 위임함으로써, 데스크톱 없이도 24시간 무결점 운영과 제어를 가능하게 한다.

### 1.2 Problem Statement
- **기존 Telegram 봇의 한계:** 일반 Telegram 앱은 메시징 중심이라 에이전트 특화 UI(승인 버튼, 폼)와 상태 표시, 컨텍스트 스레딩 등을 제대로 지원하지 못한다.
- **다중 에이전트 파편화:** 여러 에이전트를 각기 다른 봇으로 운영 시, 단일 통합 뷰(Inbox)가 없어 상태 파악과 컨텍스트 스위칭 비용이 크다.
- **모바일 제어 부재:** 기존 웹 콘솔(admin.pen 등)은 데스크톱에 최적화되어, 이동 중 긴급 알림 수신이나 즉각적인 에이전트 관리가 매우 불편하다.
- **기존 해결책의 한계:** 일반 텔레그램은 확장이 막혀 있고, 웹 UI는 모바일 접근성이 떨어짐.
- **정당화:** 익숙한 Telegram UX를 차용해 진입 장벽을 낮추면서도, 통합 Inbox와 에이전트 특화 상호작용(승인, 컨텍스트 제어)을 모바일 네이티브로 제공하는 전용 클라이언트가 필수적이다.

## 2. Target Users & Personas
### 2.1 페르소나: 1인 운영자 / 인디 해커 (Operator)
- **역할/숙련도:** 다수의 AI 에이전트를 이용해 프로덕트를 개발/운영하는 고급 사용자.
- **대상 에이전트:** Simple Claw, Open Claw (Telegram 호환 브릿지).

### 2.2 페르소나별 핵심 작업 흐름
- **일상 흐름:** PC에서는 코딩 등 딥워크, 외부 이동이나 휴식 중에는 모바일로 에이전트 상태를 모니터링.
- **트리거:** 에이전트의 긴급 알림(장애, 배포 승인 등) 수신 시, 혹은 이동 중 떠오른 태스크를 바로 지시할 때.
- **핵심 작업:** 통합 Inbox에서 여러 에이전트의 메시지를 일괄 확인하고, 원터치로 승인(Approve)하며, 텍스트/음성으로 신규 태스크를 위임.

### 3.2 핵심 UC 상세 (User Journey)
**UC-01: 긴급 시스템 장애 알림 수신 및 즉시 대응 승인**
- **트리거:** 에이전트(Simple Claw)가 서버 CPU 스파이크를 감지하고, 인스턴스 재시작 권한을 요청하는 Push 알림 발송.
- **흐름:**
  1. 시스템: Push 알림 전송 ("서버 A CPU 95% 지속 중. 자동 재시작 승인? [승인] / [거절]")
  2. 사용자: 알림 탭하여 앱 진입, 로그 컨텍스트 확인
  3. 사용자: 인라인 [승인] 버튼 클릭
  4. 시스템: 에이전트에 이벤트 전송, 로딩 표시
  5. 시스템: 처리 완료 후 결과 메시지 수신
- **성공 종료 조건:** 에이전트가 명령을 완료하고 정상화 알림 수신
- **실패/대안 경로:** 재시작 실패 시 에러 로그와 함께 "추가 진단 요청" 버튼 표시

**UC-02: 출근 중 모닝 브리핑 확인**
- **트리거:** 간밤의 시스템 운영 상태 문의.
- **흐름:**
  1. 사용자: 마이크 버튼으로 "어젯밤 이상 없었어?" 발화
  2. 시스템: STT 변환 후 에이전트로 전송
  3. 시스템: 텍스트 응답 및 TTS 오디오 반환 ("백업 크론 1개 실패")
  4. 사용자: "실패한 크론 다시 돌려줘" 텍스트 입력
  5. 시스템: 재시도 완료 피드백
- **성공 종료 조건:** 장애 파악 및 후속 조치(재시도) 완료.

**UC-03: 이동 중 아이디어 메모 및 개발 태스크 초안 위임**
- **트리거:** 산책 중 신규 기능 아이디어 지시.
- **흐름:**
  1. 사용자: "새로운 이슈 추가해줘. 제목은 다크모드..." 입력
  2. 시스템: 에이전트가 이슈 생성 툴 호출
  3. 시스템: "이슈 생성됨. 추가 검수 진행 여부?" 질문
  4. 사용자: "아니, 나중에" 입력
- **성공 종료 조건:** 시스템의 이슈로 생성 완료.

### 4.2 Non-functional Requirements
- **성능:** 콜드 스타트 2초 이내, 로컬 렌더링 지연 100ms 이내
- **보안:** API 토큰은 Secure Storage 보관, 통신은 HTTPS
- **접근성:** 스크린 리더 호환 및 WCAG AA (4.5:1) 명도 대비
- **국제화:** 한국어 우선, LTR 방향
- **안정성:** 발송 실패 시 수동 재시도, 앱 크래시 후 컨텍스트 복구
- **관측성:** 크래시 리포트 도구 연동
- **호환성:** iOS 16+, Android 12+ 지원

### 4.3 Constraints & Assumptions
1. 1차 릴리스는 단일 기기(Single Device) 기반의 로컬 저장소를 활용 (다중 사용자/디바이스 동기화 제외).
2. Cross-platform 프레임워크(React Native + Expo)를 사용하여 앱 심사 및 배포 일정을 단축한다.
3. 에이전트 백엔드 측의 호환 API 브릿지가 안정적으로 제공됨을 전제로 한다.

## 5. Technical Architecture
### 5.1 시스템 컨텍스트 다이어그램
```text
[User]
  │ (Voice / Text / Tap)
  ↓
[Agent Client (Mobile App)]
  │ (HTTPS / WSS - JSON, Audio Streaming)
  ↓
[API Gateway & Auth]
  │
  ├─→ [SimpleClaw Agent Daemon] ←→ [LLM]
  └─→ [Open Claw Agent]
```

### 5.2 기술 스택 결정
- **언어/프레임워크:** React Native (Expo) + TypeScript (크로스플랫폼 지원)
- **상태 관리:** Zustand (글로벌 상태) + React Query (API 상태 페칭/캐시)
- **음성 입출력:** Expo Speech (네이티브) 기반 후 필요시 OpenAI API
- **인증:** Token 기반 인증 (단일 운영자용 정적 토큰)
- **저장:** Expo SecureStore (민감 정보) + SQLite/AsyncStorage (메시지 캐시)
- **빌드/배포:** EAS (Expo Application Services) OTA 업데이트 활용

### 5.3 SimpleClaw 연동 인터페이스
- **API Endpoints:** 채널 목록 (`GET`), 메시지 목록 (`GET`), 송신 (`POST`), 액션 (`POST`)
- **메시지 포맷:** JSON `{ id, channel_id, sender, type: "text"|"action_request", content, metadata }`
- **통신:** Server-Sent Events (SSE) 혹은 WebSocket을 통한 타이핑 효과 및 빠른 응답 지원

### 5.4 데이터 모델
1. **Channel:** `id`, `agent_name`, `unread_count`
2. **Message:** `id`, `channel_id`, `role`, `content`, `status`
3. **ActionState:** `message_id`, `is_resolved`
4. **UserConfig:** `stt_enabled`, `theme`

### 5.5 아키텍처 위험
1. **음성 처리 Latency:** 지연 방지를 위해 로컬 네이티브 STT 최우선 적용
2. **모바일 백그라운드 제약:** FCM/APNs 연동으로 웹소켓 단절 시 알림 보완
3. **LLM 지연 (Hang):** 60초 타임아웃 및 Fallback 에러 렌더링
4. **오프라인 데이터 유실:** SQLite 큐 기반 발송 전 백그라운드 대기열 구현

## 6. Success Metrics & Scope
### 6.1 KPI 표
| 지표 | 측정 방법 | 목표값 (1차) | 목표값 (6개월) |
|------|----------|-------------|----------------|
| 일일 활성 세션 (DAU) | 앱 진입 횟수 | 3+ | 5+ |
| 알림 대응 속도 | Push 수신 후 버튼 클릭까지 | < 60초 | < 30초 |
| 응답 지연 | 입력 후 1차 응답까지 (p50) | < 4s | < 2s |
| 액션(승인) 성공률 | 버튼 클릭 대비 정상 응답 | > 95% | > 99% |
| 무중단 연속 사용 | 세션 중 Crash 없는 비율 | > 98% | > 99.5% |

### 6.2 Scope In
1. 호환 Bot API 브릿지 기반 양방향 텍스트/마크다운 메시징
2. STT/TTS 네이티브 연동
3. OS 네이티브 Push 알림 수신
4. 인라인 승인/거절 액션 버튼 렌더링 및 콜백
5. 다중 에이전트 채널 통합 Inbox 뷰

### 6.3 Scope Out
1. 다중 디바이스 메시지 클라우드 동기화
2. 그룹 채팅 및 Multi-tenant (다중 사용자)
3. 이미지/동영상 첨부 (향후 지원)
4. 복잡한 동적 서버 드리븐 UI 생성

## 8. Open Questions / Decisions Log
- **음성 인식 엔진 고도화 여부:** 네이티브 STT 품질이 한국어 IT 용어(크론, 로그, 인스턴스 등)에서 저조할 경우, 추후 Whisper API 연동을 도입할지 M3에서 실측 후 결정.
- **인증 보안 모델 (장기):** 1차에서는 Long-lived 토큰을 SecureStore에 보관하나, 향후 다중 기기 지원 시 OAuth2 토큰 회전(Rotation)을 도입해야 할 여지가 있음.
