# Agent Client — User Flow & Screen Design

> 이 문서는 PRD.md를 바탕으로, Pencil 등 화면 설계 도구가 와이어프레임/프로토타입을 생성할 수 있도록 만든 **화면 단위 설계도**다. 사람과 에이전트가 동일하게 해석할 수 있도록 ID 기반 표·이동 그래프로 기술한다.

## 0. 문서 사용 규약

### 0.1 ID 네이밍
- `S-XX` : 일반 화면(Normal Screen, 풀 페이지)
- `M-XX` : 모달/바텀시트(Modal / Bottom Sheet)
- `D-XX` : 다이얼로그/팝업/토스트(Dialog / Popup / Snackbar)
- `O-XX` : 인라인 오버레이(같은 화면 위에 떠 있는 요소, ex. 음성 입력 오버레이)
- `I-XX` : 인라인 컴포넌트(채팅 메시지 안의 카드/트레이스 패널)
- `E-XX` : 외부 화면(OS 시스템 시트, OAuth 웹뷰 등)

### 0.2 상태 표기
- `[GUEST]` : 온보딩 미완료(앱 첫 실행 ~ 최초 버디 등록 전)
- `[LOCKED]` : 온보딩 완료, 앱 잠금 활성 + 미인증 상태(패스코드/생체 잠금 화면 위)
- `[AUTH]`  : 온보딩 완료 + 잠금 해제(또는 잠금 비활성) — 일반 사용 가능 상태
- `[OFFLINE]` : `[AUTH]` 중 네트워크 단절 (Read-only 캐시 모드)
- `[ANY]`  : 상태와 무관 (Splash 등)

> **규칙**: 동일 화면이라도 상태가 달라지면 별도 ID 또는 명시적 분기(`-A`, `-G` 등)로 분리한다.

### 0.3 화면 표 형식
각 화면은 다음 필드를 가진다.

| 필드 | 설명 |
|------|------|
| ID | 위 네이밍 규약 |
| Name | 화면 표시명 |
| Type | Normal / Modal / BottomSheet / Dialog / Overlay / Inline / External |
| State | 진입 가능한 사용자 상태 |
| Info | 화면에서 노출되는 정보(데이터 항목) |
| Actions | 사용자 입력(버튼/링크/제스처) → 이동 대상 ID 또는 동작 |
| Notes | 엣지케이스, 빈 상태, 에러 처리 |

## 1. 상태 머신 (User State Machine)

```
[설치] ─► [GUEST] ──(온보딩 + 최초 버디 등록 + 패스코드 설정 옵션)──► [AUTH]
                                                                  │
                                                                  ├──(앱 잠금 활성 + 백그라운드 복귀)──► [LOCKED]
                                                                  │                                        │
                                                                  │                                        └──(인증 성공)──► [AUTH]
                                                                  │
                                                                  ├──(네트워크 단절)──► [OFFLINE]
                                                                  │                       │
                                                                  │                       └──(재연결)──► [AUTH]
                                                                  │
                                                                  └──(로그아웃/초기화)──► [GUEST]
```

전이 규칙
- `[GUEST] → [AUTH]`: S-09(온보딩 완료) 도달 시
- `[AUTH] → [LOCKED]`: 앱이 백그라운드 진입 후 임계 시간(기본 즉시) 경과
- `[LOCKED] → [AUTH]`: S-10에서 패스코드/생체 인증 성공
- `[AUTH] → [OFFLINE]`: 네트워크 감시자가 단절 감지 → D-09 배너 노출 + 송신 큐 보류
- 어떤 상태든 `[GUEST]`로 회귀: S-40 로그아웃/초기화 확정

### 4.2 출구(Back) 규칙
- 모든 `S-` 화면은 시스템 뒤로가기 / 좌상단 [←]로 직전 화면(또는 S-20)으로 복귀
- 모든 `M-`/`D-`/`O-`는 [닫기]/[취소] 또는 외부 영역 탭으로 호출 화면 복귀
- E-*는 시스템 복귀(딥링크 콜백 또는 사용자 수동)

### 4.3 데드엔드 점검 (확인 완료)
- ✅ 모든 진입 화면에서 S-20(홈) 또는 S-01(초기화 후)로 도달 가능
- ✅ `[LOCKED]`에서 패스코드 분실 케이스도 S-11 → D-06 → S-01 경로 보장
- ✅ OAuth 실패/만료는 D-13 또는 S-36에서 항상 [재인증/해제] 출구 보유
- ✅ `[OFFLINE]`에서도 S-22 입력은 큐잉, 재연결 시 자동 송신
- ✅ 외부(E-*)에서 본 앱 복귀 시 마지막 컨텍스트 유지 또는 S-20 fallback
